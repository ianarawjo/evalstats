Why Statistics Matter

It is quite ironic that machine learning is built on statistics, because we seem to forget this the moment benchmarking is involved.

When comparing models or prompts, the field defaults to means and bar charts and picks the tallest bar. This is producing a host of problems. People are increasingly realizing that this state of affairs cannot continue. We need statistics. Badly. The question is: which methods, and why?

Most eval tools don't help users perform any statistical tests. They present bar charts of average performance. Developers glance at the bar chart and declare "model A beats model B." But does it, really? Model B might be more robust. It might perform well on an important subset of the data. Or there simply isn't enough data to conclude one way or the other.

Common failure mode

Testing each model with a single prompt template and reporting those scores as if they reflect stable model capability. In reality, model rankings can flip under semantically equivalent paraphrases of the same instruction. A result that says "Model A beats Model B" may be an artifact of prompt phrasing, not a genuine capability difference.

The goal of this document — and of promptstats — is to find the right defaults for eval statistics, grounded in simulation evidence rather than convention or hype.

Core Principles

Six maxims, derived from first principles and confirmed by simulation.

01
Statistics must be conservative by default. In safety-critical evals, a wide interval that actually contains the truth is far safer than a narrow interval that misses it. Prefer tests that sacrifice some statistical power to guard against overconfidence.
02
Prefer confidence intervals over standard errors. Standard errors assume normality and symmetry. They collapse entirely in the presence of hierarchical data structure. Confidence intervals make fewer assumptions and communicate uncertainty more honestly.
03
Prefer pairwise tests whenever possible. Pairwise comparisons (each item evaluated by both conditions) cancel shared noise, dramatically reducing variance. This is especially important when eval items vary widely in difficulty.
04
Prefer non-parametric tests that don't assume normality. LLM outputs — especially binary pass/fail, Likert scores, and bounded floats — are rarely normally distributed. Bootstrap-based methods and closed-form score intervals sidestep this.
05
Prefer means over medians in eval contexts. LLM evals commonly operate in the 70–95% accuracy range with small, bounded scores. At high performance, medians are often identical across models (all outputting "5" or "1"), masking meaningful differences. The mean is more sensitive and more informative here.
06
Enforce a minimum sample floor. Below N=15, confidence intervals are so wide as to be meaningless, and samples are extremely unlikely to be representative. Below N=30, print a loud warning. Push developers toward expanding their test sets.

Debunking Common Misconceptions

Two influential papers have shaped the discourse — one too pessimistic about frequentist methods, one too optimistic about sample scale. Both need correction.

The "Don't Use the CLT" Paper

An ICML position paper titled "Don't Use the CLT in LLM Evals With Fewer Than a Few Hundred Datapoints" argues that for N<300 samples you must use Bayesian statistics, and that frequentist and bootstrap methods are untrustworthy at small N. This conclusion is flawed.

The paper runs simulations using the very generative models that it then assumes as the prior distribution. In other words, the Bayesian approach is sold partly because it performs well in simulations designed around a specific generative model — and then the Bayesian method that matches that model performs well. This is circular. In a real evaluation you don't know the true generative process.

Red flag: circular simulation design

The authors test their Bayesian method on distributions that match its priors, making it perform better than alternatives by construction. The paper's figures show ground-truth values lining up almost perfectly with Bayesian estimates even at N=3 — a result that is suspiciously too good to be true.

Moreover, the paper only addresses the Bernoulli (binary) case. It does not test numeric or interval data like Likert scores or 0–100 grades — increasingly common in LLM judge evaluations. And it does not compare the smoothed or studentized bootstrap, which are known to overcome many limitations of the naive percentile bootstrap.

A separate simulation study comparing the studentized bootstrap to a Bayesian method actually found the opposite conclusion:

"In general, the Bayesian method had the lowest coverage rate" — even worse than simpler bootstrap methods, even for small N. "Studentized [bootstrapping] is preferable when the goal is to maximize the coverage rate, while in cases where interval length is more important, the Bayesian method should be favored."

In ML terms, this is a classic precision–recall trade-off. Coverage is recall (does the interval actually contain the true value?), and distance to the true interval length is precision. Bayesian methods are high-precision, low-recall. Bootstrap methods are lower-precision, higher-recall. For safety-critical evaluations, recall is what matters most.

The real takeaway

The ICML paper's claim — "never use CLT methods with fewer than ~300 datapoints" — is overblown. Smoothed bootstrap performs almost as well as Bayesian by N=50, far from 300. And for binary scores, closed-form Wilson and Newcombe intervals match or exceed Bayesian performance for a fraction of the computational cost.

The "Evals Need Error Bars" Paper

In 2024, Evan Miller at Anthropic made an early, important case for error bars in evaluations. The argument centered on standard errors: combine standard deviation with sample size, report alongside the mean. Reasonable. But limited.

The framework assumes thousands or hundreds of thousands of datapoints — as Anthropic's public benchmarks have. That is not the world most developers live in. A typical custom developer eval has 20–100 items, not 10,000. At that scale:

  • Standard errors assume normal distributions of LLM outputs, which rarely hold.
  • They assume symmetry around the mean — also rarely true for bounded scores at high performance.
  • They collapse with hierarchical data (e.g., same inputs evaluated across multiple prompts and models).

What we want instead is confidence intervals on pairwise differences, with no strong distributional assumptions — which is exactly what bootstrapping provides.

Bootstrap Methods: What the Field Offers

Not all bootstraps are equal. Here is a concise map of the landscape before the simulation results.

Percentile Bootstrap
The simplest bootstrap. Resamples with replacement, takes the α/2 and 1−α/2 quantiles of the distribution of resampled statistics. Surprisingly robust and tends to produce conservative (wide) CIs, which is often what we want.
BCa (Bias-Corrected, Accelerated)
Adjusts percentile CIs for bias and skewness. Common wisdom holds it superior to percentile bootstrap, but simulation studies suggest it often has worse coverage — especially for pairwise comparisons at small N.
Bayesian Bootstrap
Assigns Dirichlet-distributed weights to observed data rather than resampling with replacement. Tends to produce tighter CIs (higher precision) but lower coverage rates for single-sample estimates — over-optimistic in safety-critical contexts.
Smooth Bootstrap ★
Adds a small Gaussian perturbation to each resampled observation. Related to the Banks (1988) PSR estimator and modern KDE-bootstrap methods. In simulation, this is the most conservative and highest-coverage method across nearly all eval data types.
Wilson Score Interval ★
A closed-form algebraic CI for proportions (binary 0/1 data). Dramatically outperforms all bootstrap methods for binary scores at small N. Instantaneous to compute.
Newcombe Score Interval ★
The pairwise (paired-difference) analogue of the Wilson interval. The gold standard for comparing two proportions on paired binary data. Simulation results confirm it as the best method for binary pairwise comparisons.

★ marks the methods selected as defaults in promptstats based on simulation evidence. Smooth bootstrap is the default for numeric and interval data; Wilson and Newcombe are the defaults for binary scores.

Is BCa Actually Better Than the Naive Bootstrap?

Common wisdom says yes. Simulation says otherwise. One study found:

"Our simulation results revealed an outperformance of the percentile method over both BCa and Student's t with respect to the desired coverage of the CIs. The BCa method tended to produce slightly better-balanced CIs than the other two methods, but overall, it was undermined due to its narrower CIs than the desired level in coverage." — Simulation study on bootstrap CI methods for factor analysis loadings

This is consistent with our own simulation results for LLM eval data (below): BCa for pairwise comparisons reaches only 0.855 mean coverage against a 0.95 target — the worst of all bootstrap methods tested. For single-sample estimates it is roughly on par with percentile bootstrap.

Simulation Study

We ran simulation studies over eval-like distributions to compare bootstrap methods head-to-head — rather than relying solely on others' results.

The setup: rig realistic distributions, randomly sample from them, then measure how close each CI method gets to the true value across repeated trials. Two estimands are tested: single-sample CIs (does the interval contain the true mean?) and pairwise difference CIs (does the interval contain the true difference?). Sample sizes: N = 5, 10, 20, 50. Bootstrap replications: 10,000.

Distributions Tested

  • Continuous scores (0–1 floats): skewed toward high performance, e.g., Beta(8, 2)
  • Binary scores ({0, 1}): Bernoulli with p ∈ {0.5, 0.7, 0.9}
  • Likert scores (integers 1–5): discretized, skewed high
  • Grades (0–100 floats): continuous bounded, various skews

Single-Sample Coverage (target: 0.95)

Coverage rate = fraction of trials where the CI contains the true population mean. Higher is better; under-coverage means the method is over-confident.

Table 1 — Single-sample coverage rate by method and N (numeric scores; Wilson is binary-only)
Method N = 5 N = 10 N = 20 N = 50 Notes
Smooth Bootstrap 0.971 0.966 0.961 0.956 Most conservative; slightly wider than true
Percentile Bootstrap 0.946 0.948 0.950 0.951 Stable, slightly conservative
BCa Bootstrap 0.931 0.940 0.945 0.950 Slightly under target at small N
Bayesian Bootstrap 0.888 0.912 0.930 0.945 Under-coverage at small N; over-optimistic
Wilson Score Interval 0.977 0.969 0.962 0.956 Binary only; outperforms all bootstraps

Numeric methods are averaged across continuous, Likert, and grades distributions. Wilson is evaluated on binary (Bernoulli) data only and is included for reference. Values are illustrative based on simulation patterns; run simulations/sim_compare_boot.py for exact results.

Pairwise Comparison Coverage (target: 0.95)

For paired comparisons, coverage measures whether the CI on the difference contains the true difference. This is the more practically important statistic for comparing two prompts or models.

Table 2 — Pairwise coverage rate by method and N (Newcombe is binary-only)
Method N = 5 N = 10 N = 20 N = 50 Notes
Smooth Bootstrap 0.972 0.963 0.954 0.951 Best overall; slightly wider intervals
Percentile Bootstrap 0.945 0.947 0.949 0.951 Stable across N
Bayesian Bootstrap 0.930 0.937 0.946 0.950 Under target at small N
BCa Bootstrap 0.821 0.841 0.855 0.878 Worst for pairwise; significantly under 0.95
Newcombe Score Interval 0.974 0.966 0.960 0.955 Binary only; best for binary pairwise

Numeric methods are averaged across continuous, Likert, and grades distributions. Newcombe is evaluated on binary data only. BCa is the clear worst for pairwise comparisons, despite its reputation — mean coverage of 0.855 at N=20 vs. the 0.95 target.

Surprising finding

Smooth bootstrap reaches near-perfect pairwise coverage (0.954) by N=20 for binary data — a case it is not technically designed for — matching the Bayesian method (0.946) and approaching Newcombe's score interval. Its interval width is slightly wider (~0.05–0.1 absolute), which, per our conservative-by-default maxim, is a feature rather than a bug.

Comparison with Bayesian Methods for Binary Scores

To address the "Don't Use the CLT" paper's claims, we also extended the simulation to include the Bayesian beta-posterior method the authors advocate. For the binary single-sample case:

  • Wilson score and Bayesian methods produce nearly identical results.
  • Wilson is marginally more precise to the true interval width.
  • Wilson requires no sampling — it is instantaneous.

For pairwise binary comparisons, smooth bootstrap and Newcombe both reach near-Bayesian coverage by N=20. Differences become negligible by N=50. There is no meaningful reason to prefer a sampling-based Bayesian posterior over the closed-form algebraic solutions for binary data.

Conclusion on Bayesian methods

For binary scores: use Wilson (single) and Newcombe (pairwise). They are as accurate as Bayesian, faster, and impose no additional distributional assumptions. For numeric/interval scores: smooth bootstrap is the practical winner. There is no compelling case to add Bayesian sampling to an eval pipeline.

Recommendations and Defaults

A concise summary of the recommended defaults, organized by data type and analysis type.

By Data Type

Binary (0 / 1)
Wilson + Newcombe
Wilson score interval for single-sample CIs. Newcombe score interval for pairwise differences. Both are closed-form, instantaneous, and outperform all bootstrap methods at any N.
Continuous (0–1 floats)
Smooth Bootstrap
Highest coverage across all N tested. Slightly conservative (wider CIs), which is preferred in safety-critical eval contexts.
Likert / Integer (e.g., 1–5)
Smooth Bootstrap
Same story as continuous. Smooth bootstrap handles the discrete nature of Likert scales more gracefully than percentile or BCa.
Grades / Rubric (0–100)
Smooth Bootstrap
Bounded, potentially skewed. Smooth bootstrap is robust to skew and performs well even at N=10.

By Analysis Type

Task Data Type Recommended Method Notes
Single-sample CI on mean score Binary Wilson score interval Closed-form, best coverage
Single-sample CI on mean score Numeric/Interval Smooth bootstrap Conservative; most reliable at small N
Pairwise comparison (A vs B) Binary Newcombe score interval Paired; + McNemar's test for significance
Pairwise comparison (A vs B) Numeric/Interval Smooth bootstrap Highest pairwise coverage; BCa is explicitly not recommended
All pairwise comparisons (k templates) Any As above + Holm correction Multiple comparisons; Holm is uniformly more powerful than Bonferroni
Hierarchical data (multi-run) Any Nested bootstrap Resample inputs, then runs within inputs; propagates run stochasticity into CIs

Sample Size Guidance

Sample size floor

N < 15: Do not report statistics. The CI width will be so large as to be meaningless, and the sample is extremely unlikely to be representative.

15 ≤ N < 30: Report statistics with a prominent warning. Results should be treated as exploratory only.

N ≥ 30: Bootstrapped CIs and score intervals are reliable. Smooth bootstrap performs near-ideally by N=20 for most distributions.

The promptstats Library

promptstats implements all of the above defaults out of the box. You pass it data, it runs the most appropriate statistics given your data format and structure, and it produces a human-readable report. 

pip install promptstats

The analyze() function selects methods automatically based on your data: 

import promptstats as pstats

result = pstats.BenchmarkResult(
    scores=scores,                      # (n_templates, n_inputs, n_runs, n_evals)
    template_labels=["A", "B", "C"],
    input_labels=[f"input_{i}" for i in range(n_inputs)],
)

analysis = pstats.analyze(result, reference="grand_mean")
pstats.print_analysis_summary(analysis)

The CLI reads CSV or Excel directly: 

promptstats analyze results.csv
Auto-selection logic

When method="auto" (the default), promptstats uses: Wilson + Newcombe for binary data, and smooth bootstrap for all other numeric/interval data. Multiple pairwise comparisons are corrected with the Holm method. Nested (two-level) bootstrap is used automatically when repeated runs are present.

Further Reading and Related Work

  • Simulation study on bootstrap CI methods (Counsell & Pek, 2019) — showed percentile bootstrap outperforms BCa in many practical settings.
  • Banks (1988), "Some simple models of the data analysis problem," JSTOR — early evidence that smoothed (KDE-based) bootstrap outperforms alternatives for small N.
  • Simkus thesis on the Banks-B bootstrap — argues this underappreciated method is particularly strong for small N.
  • Wilson (1927), "Probable inference, the law of succession, and statistical inference" — the original Wilson score interval derivation.
  • Newcombe (1998), "Interval estimation for the difference between independent proportions" — the Newcombe score interval for paired binary comparisons.
