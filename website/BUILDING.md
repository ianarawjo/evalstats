# Website Build System

All HTML is generated by `python website/build.py`. Do not hand-edit the output files.

## Source layout

```
website/
  src/                  ← page body content (one file per top-level page)
    index.html
    choose.html
    resources.html
    principles.html
    roadmap.html
    ...

  partials (inline in build.py):
    _NAV_LINKS          ← canonical top-nav links — edit to add/remove nav items
    FOOTER_HTML         ← shared footer
    make_head()         ← shared <head> (fonts, dark-mode detect, CSS link)

  index.css             ← styles for index.html (extracted from inline <style>)
  choose.css            ← styles for choose.html
  inv.css               ← styles for investigation + article pages
  nb.css                ← additional styles for notebook content
  dark.js               ← dark-mode toggle (shared, loaded at end of <body>)

  notebooks/            ← Jupyter notebooks for investigation pages
  investigations/       ← generated investigation pages (build artifacts)
  gen_stubs.py          ← investigation metadata + left sidebar generator
```

## Two page types

### `"full"` (index, choose)
src file = everything between the nav and footer.
Wrapping: `head + nav + src_content + footer + dark.js`

### `"article"` (resources, principles, roadmap, …)
src file = just the `<article>` body content.
Wrapping: `head + nav + inv-header + page-layout(sidebar + article) + footer + dark.js`
The left sidebar is generated by `gen_stubs.make_nav(active_sidebar, prefix="./")`.

## Adding a new top-level page

1. Create `website/src/<slug>.html` with body content only (no `<html>`, nav, or footer).
2. Add an entry to `PAGE_CONFIGS` in `build.py` with `slug`, `title_tag`, `type`, `css`, and any article-type fields (`eyebrow`, `subtitle`, `active_sidebar`).
3. Run `python website/build.py`.

## Changing the top navigation

Edit `_NAV_LINKS` in `build.py` — one list, propagates to every page including investigation pages.

## Investigation pages

Defined in `gen_stubs.py` (`INVESTIGATIONS` list).
Built by `build.py`: notebook → full page, or stub if no notebook exists.
Left sidebar (`make_nav()`) now accepts `prefix` ("../" for investigations, "./" for top-level pages).

## Build commands

```bash
python website/build.py               # rebuild everything
python website/build.py --pages       # top-level pages only
python website/build.py model-vs-model  # one investigation
python website/build.py resources     # one top-level page
python website/build.py --execute     # also run notebooks
```
