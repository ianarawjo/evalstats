The promptstats Workflow

Three steps, no statistics PhD required.

pip install promptstats

CLI Quick Reference

The fastest path to results: drop your CSV in, get statistics out. No Python required.

Basic usage

# Run full analysis — auto-detects data type and structure
promptstats analyze results.csv

# Brief executive leaderboard only (fastest read)
promptstats analyze results.csv --brief

# 95% confidence intervals instead of the default 99%
promptstats analyze results.csv --ci 0.95

Saving output

# Save Markdown summary + advantage plot
promptstats analyze results.csv --out report.md plot.png

# Save full structured results as JSON (for programmatic use)
promptstats analyze results.csv --out analysis.json

# Brief terminal + full Markdown saved to disk
promptstats analyze results.csv --brief --out report.md

Excel files

# First sheet (default)
promptstats analyze results.xlsx

# Named sheet
promptstats analyze results.xlsx --sheet "Eval Results"

# Sheet by 0-based index
promptstats analyze results.xlsx --sheet 2

Choosing a method

# Default: auto-detect (smooth bootstrap for numeric, Wilson for binary)
promptstats analyze results.csv

# Force mixed-effects model (LMM) — accounts for input-level random effects
promptstats analyze results.csv --method lmm

# More bootstrap resamples for higher precision (default: 10000)
promptstats analyze results.csv --n-bootstrap 20000

Reference baseline & advanced flags

# Compare all prompts against a specific baseline instead of the grand mean
promptstats analyze results.csv --reference "zero-shot"

# Flag inputs scoring below a threshold in the robustness table
promptstats analyze results.csv --failure-threshold 0.5

# Show top 10 pairwise comparisons (default: 5)
promptstats analyze results.csv --top-pairwise 10

# Run the Friedman omnibus test before pairwise comparisons
promptstats analyze results.csv --omnibus

# Full option list
promptstats analyze --help

Input Format

promptstats accepts two CSV/Excel layouts: wide (one column per prompt template) and long/tidy (one row per observation). Long format is more flexible and recommended for multi-model or multi-run data.

Wide format

Rows are inputs; columns are prompt templates (or models). Fastest to create from a spreadsheet view.

input zero-shot few-shot chain-of-thought
q_0010.720.850.91
q_0020.600.780.83
q_0030.880.900.87
…………

Long / tidy format

One row per observation. Required for multi-model data. The input column links paired observations across prompts/models so that paired statistical tests can account for per-input difficulty differences.

model prompt input score
gpt-4o-mini5-shotq_0010.85
gpt-4o-mini0-shotq_0010.61
gemma-3-4b5-shotq_0010.77
gemma-3-4b0-shotq_0010.60
…………
Column name aliases

Column names are matched case-insensitively and a range of aliases are accepted: prompt / template / prompt_template — input / example / item / id — score / value / result / metric — model / model_name / model_label. For repeated runs, add a run (or seed / trial) column.

Multi-run format

When the same input is evaluated under the same prompt/model multiple times (e.g., at different seeds), add a run column. With ≥ 3 runs per cell, promptstats switches to a nested bootstrap that accounts for seed-level variance on top of input-level variance — giving you more honest uncertainty estimates.

model prompt input run score
gpt-4o-mini5-shotq_00100.85
gpt-4o-mini5-shotq_00110.82
gpt-4o-mini5-shotq_00120.88
……………

Python API Quick Reference

All functions return structured result objects with a .summary() method that prints the full terminal report.

Comparing prompt templates

import numpy as np
import promptstats as pstats

scores = {
    "zero-shot":        np.array([0.72, 0.60, 0.88, 0.75, ...]),
    "few-shot":         np.array([0.85, 0.78, 0.90, 0.82, ...]),
    "chain-of-thought": np.array([0.91, 0.83, 0.87, 0.95, ...]),
}

result = pstats.compare_prompts(scores)
result.summary()          # full terminal report
result.quick_summary()    # one-line verdict per template

Comparing models

scores = {
    "gpt-4o-mini":   np.array([0.82, 0.91, 0.78, ...]),
    "gemma-3-4b-it": np.array([0.75, 0.84, 0.71, ...]),
    "qwen3-8b":      np.array([0.88, 0.93, 0.85, ...]),
}

result = pstats.compare_models(scores)
result.summary()

Models × prompts grid

# Nested dict: outer = models, inner = prompt templates
scores = {
    "gpt-4o-mini": {
        "5-shot": np.array([0.79, 0.85, 0.83, ...]),
        "0-shot": np.array([0.61, 0.73, 0.69, ...]),
    },
    "gemma-3-4b-it": {
        "5-shot": np.array([0.68, 0.77, 0.72, ...]),
        "0-shot": np.array([0.60, 0.72, 0.65, ...]),
    },
}

result = pstats.compare_models(scores)
result.summary()

Analyze from a CSV / Excel file

import promptstats as pstats

# Mirrors the CLI — auto-detects format and data type
result = pstats.analyze("results.csv")
result.summary()

# With options
result = pstats.analyze(
    "results.xlsx",
    sheet="Eval Results",
    ci=0.95,
    method="lmm",        # mixed-effects model
    n_bootstrap=20000,
)
result.summary()

Brief executive summary

import promptstats as pstats

# After any analysis, print only the leaderboard
result = pstats.analyze("results.csv")
pstats.print_brief_summary(result)   # entity · group · mean · CI · verdict

Common options (all functions)

Parameter Default Notes
ci0.99Confidence level. Use 0.95 for the traditional default.
method"auto"Auto-selects based on data type. Options: "lmm", "wilson", "newcombe", "bootstrap", "smooth_bootstrap", "bca", "bayes_bootstrap".
n_bootstrap10000Bootstrap resamples. Increase for smoother CI estimates.
correction"fdr_bh"Multiple-comparison correction. Options: "holm", "bonferroni", "fdr_bh", "none".
simultaneous_ciTrueFamily-wise CI correction (max-T). Set to False for marginal CIs.
statistic"mean"Central tendency. Also supports "median".
reference"grand_mean"Advantage plot reference. Pass a template label to compare against a specific baseline.
omnibusFalseRun Friedman omnibus test before pairwise comparisons (k ≥ 3).

Reading the Output

The terminal report has four main sections, printed in order:

Mean Performance
ASCII forest plot showing each entity's mean score with marginal bootstrap confidence interval. The │ marker shows the grand mean. Wider bars = more uncertainty.
Pairwise Comparisons
Table of all entity pairs, sorted by CI distance from zero. Includes the difference CI and a significance verdict after multiple-comparisons correction.
Rank Probabilities
Bootstrap-estimated probability that each entity holds each rank position (1 = best). Wide distributions across ranks signal ties or insufficient data.
Executive Summary ★
The bottom leaderboard: entities grouped by statistical significance (same letter = no clear difference). Also the only section printed by --brief.
Significance groups

Entities sharing the same letter label (A, B, AB, …) are not statistically distinguishable after multiple-comparisons correction. If two entities both show label A, the data can't rule out that they perform the same — even if one has a higher mean score.
